# VideoSeal Watermarking Test Suite

欢迎使用 VideoSeal 专业级盲水印测试套件。本目录包含了一套基于深度学习（PixelSeal 模型）和 BCH 纠错码（Bose-Chaudhuri-Hocquenghem）算法的端到端图像/视频盲水印测试工作流。

该测试套件实现了**高鲁棒性**的盲水印注入和提取，主要核心能力包含：
1. **隐写植入**：基于频率或像素域微小摄动技术，在保持图像肉眼高保真的情况下，将加密载荷（如溯源 ID 或字符串）嵌入图片。
2. **纠错与恢复 (ECC)**：通过自动寻找并引入最优参数的 BCH 校验码方案，即使图像残差在攻击中遭受大量误码翻转，亦能利用冗余空间进行错误纠正，从而准确还原原始数据。
3. **全面对抗与鲁棒性验证**：集成了十多种经典的图像单项与复合破坏性攻击测试，涵盖 JPEG 压缩、裁剪缩放、旋转变形、模糊加噪、亮度/饱和度变异等。

为方便测试，所有的脚本**均已解耦命令行传参**。用户可直接在对应 Python 脚本的 `USER CONFIGURATION` 配置区修改参数并直接执行。

---

## 🛠 核心工作流及命令

环境要求：`conda activate videoseal`

标准测试工作流通常按照以下顺序执行：

### 1. `init_env.py` (环境探针与基础硬件诊断)
用于验证基础框架、模型权重下载和硬件加速（CUDA/MPS）的就绪状态。在首次配置环境时，可用于检测深度学习模型能否正常加载并进行推理。
- **支持自动检测的模型**：`videoseal`, `pixelseal`
- **使用方法**：
  在代码顶部的 `Config` 类中修改 `MODEL_NAME` 和 `FORCE_DEVICE`，然后运行：
  ```bash
  python init_env.py
  ```

### 2. `bch_config_generator.py` (BCH 纠错参数分配与生成)
水印系统单次可写入的总物理容量受限于模型结构（固定为 256 bits）。
本脚本**基于用户需要写入的“最大信息长度”，在数学上推演出最优的本原多项式，并将剩余的比特容量全部分配给纠错码 (ECC) 空间。**
*核心原则：系统预设的水印长度上限越短，ECC 可获得的空间余量越大，水印的抗破坏容错能力呈指数级提升。*
- **使用方法**：
  在代码顶部的 `UserConfig` 类中修改：
  - `STRENGTH`：水印嵌入物理强度 (推荐取值 `1.5` - `2.5`)
  - `MAX_WATERMARK_LENGTH`：系统允许嵌入的**字符长度上限**
  - `CHARSET`：字符集字典。系统会通过 `log2(len(charset))` 自动评估单字符所占比特带宽。
  配置完成后运行：
  ```bash
  python bch_config_generator.py
  ```
  *(脚本将自动在目录下生成全局配置文件 `bch_config.json`，供后续流程读取)*

### 3. `demo.py` (水印注入与提取引擎)
依赖 `bch_config.json` 的架构配置，读取指定的原始图像进行神经网络不可见水印注入。在验证完成闭环提取后，将图像以高质量 JPEG 格式保存。
- **使用方法**：
  在代码下半部分的 `USER CONFIGURATION` 区域配置：
  - `INPUT_PATHStr`：需注入水印的源图片路径或目录
  - `WATERMARK_TEXT`：实际注入的水印字符串文本
  配置完成后运行：
  ```bash
  python demo.py
  ```
  *(生成的文件默认将输出至原目录同级、带有 `_wm` 后缀的新文件夹中)*

### 4. `test_attacks.py` (对抗与鲁棒性自动化测试)
对已注入水印的图像应用多种干扰与破坏算法，以模拟实际互联网传播中面临的信道劣化情况。
系统将在终端输出标准化的鲁棒性测试报告，并记录各项攻击下 BCH 引擎成功修正的错误位数 (`ECC recovered bits`)。
- **使用方法**：
  在代码下半部分的 `USER CONFIGURATION` 区域配置：
  - `TEST_IMAGE_PATHStr`：指向需要测试的水印图像路径 (例如 `val2017_subset_wm/000000001675_wm.jpg`)
  *(注：该脚本会自动引用 `demo.py` 中配置的预期字符串 `WATERMARK_TEXT` 进行一致性比对)*
  运行：
  ```bash
  python test_attacks.py
  ```

## 💡 最佳实践与鲁棒性调优指南

如果在对抗攻击测试中存在较多失败项（例如在高斯噪声叠加高度 JPEG 压缩的复合攻击下无法提取），表明当前水印的信息密度过高、导致用于 ECC 的冗余比特不足以覆盖信道误码。建议采取以下策略进行参数调优：

1. **降低长度上限设定**：修改 `bch_config_generator.py`，适度**调低** `MAX_WATERMARK_LENGTH` (例如对于常见的业务溯源码可设为 `10-16`)。该操作会将极大的比特空间释放给 ECC 分配机制，从而显著提升纠错极限。
2. **提高水印嵌入强度**：在 `bch_config_generator.py` 中，适度提高 `STRENGTH` 参数 (推荐设为 `2.0 - 2.5`)。此举能提升嵌入频域的高频残差信噪比，增强对抗空间几何变换和压缩滤波的能力。
3. **完成配置后，重置测试管线**：
   在调整完底层参数后，请确保依次执行以下指令使配置生效于最终图像：
   ```bash
   python bch_config_generator.py # 1. 重新计算并刷新 BCH JSON 配置
   python demo.py                 # 2. 依照新版配置重新生成并注入水印图像
   python test_attacks.py         # 3. 读取新的水印图像，执行对抗鲁棒性测试
   ```
